# MCP 接入

AniLinkServer 支持通过后台“**MCP接入**”页面快速复制客户端所需的配置 JSON。

## 访问入口

1. 登录管理后台。
2. 进入左侧菜单中的“**MCP接入**”。
3. 在页面上可以看到当前站点对应的 `MCP API Key` 和 `MCP 端点 URL`。
4. 复制页面中的 **配置 JSON**，然后粘贴到你的客户端中使用。

## 复制配置 JSON

你可以直接复制页面中的 JSON 配置内容，该配置通常包含以下字段：

- `transport`：传输方式，例如 `streamable_http`
- `url`：MCP 服务器地址
- `headers`：包含 `X-API-KEY`

示例配置：

```json
{
  "transport": "streamable_http",
  "url": "http://你的站点域名或IP:端口/api/v1/mcp",
  "headers": {
    "X-API-KEY": "你的-api-key"
  }
}
```

## 关键注意事项

- **站点 URL 必须配置正确**。
- 如果你使用域名或反向代理，请确保 `url` 中填写的是外部可访问的地址。
- 如果你的服务部署在内网，请确保客户端能够访问该 URL。
- 站点 URL 中的端口必须与实际暴露的端口一致。
- 如果页面上提供了 “复制配置 JSON” 按钮，优先使用该方式复制，避免手工填写出错。

## 常见问题

- **为什么客户端无法连接？**
  - 请检查 `url` 是否为实际可访问地址。
  - 请检查 `X-API-KEY` 是否与后台显示一致。
  - 如果使用 HTTPS，请确保证书可被客户端信任。

- **我应该复制哪一部分？**
  - 直接复制“配置 JSON”中的完整内容最稳妥。
  - 如果客户端只支持单独填写 URL 和 API KEY，也可以分别填入对应字段。

## 适用场景

该功能适用于需要通过第三方客户端接入 AniLinkServer 后端的场景，例如使用支持 MCP 协议的客户端进行视频播放和远程控制。
