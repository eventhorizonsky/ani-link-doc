# 下载管理

下载管理用于集中查看所有资源下载任务的实时状态，并提供取消、重试、删除和绑定排查能力。

## 功能入口

进入后台后，在“资源下载（实验性功能）”分组中选择“下载管理”。

## 页面说明

下载管理页面会展示当前全部下载任务，并通过 SSE 实时更新。

常见能力包括：

- 查看任务进度
- 手动刷新任务列表
- 取消任务
- 重试任务
- 删除任务记录
- 查看任务绑定状态

## 任务状态

虽然具体状态值由后端决定，但通常可以按以下思路理解：

- 进行中：任务正在下载
- 已完成：资源已经下载完成，正在或已经入库
- 已取消：用户手动停止
- 失败：下载过程或后处理过程出错

建议结合绑定状态一起判断任务是否真正完成。

## 刷新任务

正常情况下，页面会自动接收进度推送；如果发现列表长时间不更新，可以手动点击“刷新任务”。

如果刷新后仍无变化，可能需要进一步检查：

- SSE 是否中断
- 下载器是否仍在工作
- 资源节点是否可用

## 取消任务

当任务处于下载中且不再需要时，可以执行取消。

适用场景：

- 选错资源
- 下载速度过慢，准备换源
- 目标媒体库选择错误

## 重试任务

失败任务可以直接重试，适合临时性问题，例如：

- 网络波动
- 资源节点短暂异常
- 下载器暂时不可用

如果重复失败，建议先检查下载配置，而不是连续多次重试。

## 删除任务

删除通常只移除任务记录，不代表一定会回滚已下载文件。

操作前建议确认：

- 任务是否已经无参考价值
- 是否已经有新的替代任务
- 是否仍需保留绑定状态用于排错

## 查看绑定状态

绑定状态窗口是排查下载后入库问题的关键页面。

可重点关注：

- `最终路径`：确认文件实际保存到了哪里
- `媒体文件 ID`：确认是否生成了媒体记录
- `绑定结果`：确认文件是否已进入媒体库
- `动画标题 / 剧集标题`：确认识别是否正确
- `匹配状态`：确认是否成功匹配到目标剧集

## 与其他功能的关系

- 若任务创建来源于手动搜索，请结合“资源搜索”文档使用。
- 若任务来自自动拉取，请结合“RSS订阅”文档排查。
- 若任务完成但未正常显示在媒体库中，请继续查看“媒体库管理”和“视频资源管理”。

## 常见问题

### 任务一直不动

- 检查下载器配置是否完整。
- 检查临时下载目录是否可写。
- 检查下载并发是否过低或被限速。

### 已完成但媒体库里没有文件

- 打开绑定状态查看最终路径和媒体文件 ID。
- 再到媒体库管理中确认目标库路径是否正确。
- 如有必要，手动触发媒体库重新获取元数据。
